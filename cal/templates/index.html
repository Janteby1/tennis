How We Did It

Before we were able to collect all our data, we first had to know what data was available, accessible and scalable. This led us to census data, data collected at the state and city level, as well as various API'sincluding Google Maps, Yelp, and Streeteasy. One key requirement, apparent from the outset, was to reference any data we collected to a particular NYC neighborhood. That meant storing data categorized at the neighborhood level. However, more often than not, data available was by geographical coordinates or zip code, leading us to create a mapping to the appropriate neighborhood.

Having all our data reference a particular neighborhood simplified the database and algorithm design processes. We stored our data in a star schema design with our neighborhoods at the center, this enabled quick quering as well as the ability to add, remove, and update our data seamlessly. Our algorithm uses this data to identify neighborhoods whose characteristics most closely match what a user inputs inputs in our form, finding the best suited neighborhoods across NY. The algorithm weights more heavily what is important to users, to ensure suitable responses.

Lastly, we were able to add other features like map and street view, learn more and the ability to graph all results from a query to find trends. Along with Facebook login, these features help create a conprehensive and user friendly site for anyone to use.
